# extract CLI

`extract` is used for working with Markdown files – it extracts image placeholders and metadata from front-matter. Logic has been extracted to the `internal/extractor` module.

## Running

```bash
# one-time execution
go run ./cmd/extract <subcommand> [options]

# build binary
go build -o extract ./cmd/extract
./extract <subcommand> [options]
```

## Subcommands

| Subcommand     | Description | Key flags |
|----------------|-------------|-----------|
| `images`       | Extracts `{{IMAGE_X}}` placeholders from Markdown and saves JSON with metadata. | `-input`, `-output` |
| `frontmatter`  | Parses YAML front-matter into CMS structure and saves JSON. | `-input`, `-output` |

### images

```bash
go run ./cmd/extract images \
  -input content/article.md \
  -output output/images.json
```

Result is an array of `Image` objects (see `internal/extractor/types.go`) with filled fields: `alt`, `prompt`, `style`, `aspect_ratio`, `position_in_article`.

### frontmatter

```bash
go run ./cmd/extract frontmatter \
  -input content/article.md \
  -output output/cms.json
```

Function:
- Reads YAML fragment between `---` and `---`.
- Generates slug if missing in data.
- Searches for FAQ section (`## FAQ` or `## Frequently Asked Questions`) and adds it to the `schema_org` field.

## Error Handling

Missing required flags (`-input`, `-output`) exits with code `1`. All error messages go to stderr.

## Data Structures

- `internal/extractor/types.go` – structure definitions for `Image`, `CMSData`, `SchemaOrg`, `FAQItem`.
- `internal/extractor/images.go` – placeholder extraction logic.
- `internal/extractor/frontmatter.go` – front-matter and FAQ parsing.

JSON files generated by CLI use two-space indentation (`  `) for readability.
